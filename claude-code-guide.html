Claude Code 入門教學

使用老師提供的 API Key 在 Windows 上安裝並開始使用 Claude Code

ℹ
本講義重點

Claude Code 在 2025 年底推出原生 Windows 版，不再需要安裝 WSL。本講義採用「老師提供 API Key」的方式，讓你用老師的帳號額度連接 Claude，不需要自己註冊付費帳號。

⚠
API Key 使用守則（請務必遵守）
  • API Key 是老師的費用，每次使用都要計費，請勿浪費
  • 不可將 API Key 分享給班級以外的人
  • 不可用於與上課無關的用途（打遊戲、寫作業以外的事）
  • 違規者將立即停止 API Key 使用權限

安裝 Git for Windows（必要）

⚠

Claude Code 內部需要 Git Bash 才能運作，即使你不使用 Git 也必須安裝。若跳過此步驟，Claude Code 會無法正常執行。

  1. 開啟瀏覽器，前往 Git 官方網站

    網址：https://git-scm.com/downloads/win

  2. 下載 64-bit Git for Windows

    點選「64-bit Git for Windows Setup」下載安裝檔

  3. 執行安裝，全部使用預設值

    一路按 Next，不需要更改任何選項。確認「Add Git to PATH」有勾選（預設已勾）。

  4. 安裝完成後重開 PowerShell

    舊的 PowerShell 視窗不會生效，必須關閉後重新開啟。

✓

驗證安裝：在 PowerShell 輸入 git --version，若出現版本號代表安裝成功。

安裝 Claude Code（原生安裝器）

方法 A：PowerShell 安裝 官方推薦

最簡單，自動更新，不需要管理員權限

方法 B：WinGet 安裝 可選

Windows 11 內建，需手動更新

方法 A：PowerShell 安裝（推薦）

  1. 以一般使用者身分開啟 PowerShell

    按 Win 鍵 → 搜尋「PowerShell」→ 直接開啟（不需要以系統管理員執行）

  2. 輸入以下安裝指令並按 Enter
    複製貼上 irm https://claude.ai/install.ps1 | iex

    等待下載並安裝完成（約 1–2 分鐘）

  3. 關閉 PowerShell，重新開啟

    PATH 設定需要新視窗才會生效

  4. 驗證安裝
    claude --version

    看到版本號（例如 2.1.xx (Claude Code)）代表安裝成功

方法 B：WinGet 安裝（備用）

# Windows 11 / Windows 10 22H2+ 適用
winget install Anthropic.ClaudeCode
⚠

WinGet 安裝後不會自動更新，需定期手動執行 winget upgrade Anthropic.ClaudeCode。

4

設定 API Key（最重要的步驟）

ℹ
為什麼用 API Key 而非登入帳號？

老師提供 API Key 讓同學使用，不需要每個人自己購買 Claude 訂閱。設定環境變數後，Claude Code 會自動使用 API Key 連接，不需要瀏覽器登入。

方式一：為目前這次 PowerShell 視窗設定（暫時）

# 將下面的 sk-ant-api03-xxxx 換成老師給你的 API Key
$env:ANTHROPIC_API_KEY = "sk-ant-api03-你的金鑰內容"

缺點：關閉 PowerShell 後設定就消失，每次都要重新輸入。

方式二：永久設定環境變數（推薦）

  1. 在 PowerShell 輸入以下指令
    # 將 sk-ant-api03-xxxx 換成老師給你的 API Key
    [System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-api03-你的金鑰內容", "User")
  2. 關閉並重新開啟 PowerShell

    環境變數設定需要新視窗才能生效

  3. 驗證是否設定成功
    echo $env:ANTHROPIC_API_KEY

    應該要顯示你的 API Key 內容

⚠
注意：API Key 計費說明
  • 設定環境變數後，Claude Code 會用 API Key 計費，不是老師的訂閱額度
  • 每次使用 Claude Code 都會消耗 token（依照對話長度計費）
  • 請不要做沒有意義的大量測試，避免浪費老師的費用

常見問題排除

❌ 出現「claude: command not found」或「claude 不是內部或外部命令」

PATH 沒有更新。解決方式：關閉並重新開啟 PowerShell 視窗。如果還是不行，手動加入 PATH：

[System.Environment]::SetEnvironmentVariable("PATH", "$env:PATH;$env:USERPROFILE\.local\bin", "User")
❌ 出現「Authentication required」或要求登入

API Key 沒有設定成功。請確認環境變數有設好：

echo $env:ANTHROPIC_API_KEY

如果沒有顯示 Key，重新執行步驟 4。

❌ 出現「API error」或「rate limit exceeded」

API Key 使用量過高或老師暫停了 Key。請告知老師，等候重置。

❌ 安裝指令出現「The token '&&' is not a valid statement separator」

你在 PowerShell 裡輸入了 CMD 的指令。確認你開的是 PowerShell（視窗標題有 PS 字樣），再重新輸入安裝指令。

❌ 防毒軟體跳出警告

Claude Code 的 .exe 檔案有時會被防毒誤判。可以將 ~\.local\bin\claude.exe 加入防毒軟體的排除清單，二進位檔案直接來自 Anthropic 官方伺服器。
